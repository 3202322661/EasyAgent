## 🛠️ Claude Code UI 核心设计哲学

* **输入框即是一切：** 减少复杂的侧边栏配置，一切动作通过自然语言或斜杠命令（`/cmd`）触发。
* **状态绝对透明：** 智能体在干嘛（思考、调工具、报错、卡住），必须有实时、细腻的动态反馈。
* **非阻塞异步体验：** 本地工具（如 PaddleOCR, Bash 执行）非常耗时，界面绝对不能假死。
* **富文本打印：** 代码块、状态标志、报错信息需要有极其强烈的色彩对比（暗色调为主）。

## 📘 Claude Code 智能体 UI 界面设计指导手册

### 🧱 一、 布局拓扑结构 (Layout Architecture)

界面整体采用两栏或单栏极简纵向布局：

### 🎨 二、 视觉与色彩规范 (Themes & Typography)

推荐采用 **Monokai / Cyberpunk 暗黑科技风**，高对比度的配色能让大模型输出的代码和报错一目了然：

| 元素组件 | 推荐配色 (HEX) | 视觉心理作用 |
| --- | --- | --- |
| **主背景色 (Background)** | `#1E1E1E` 或 `#121212` | 极客极简，减少眼睛疲劳 |
| **用户提问 (User Text)** | `#FFFFFF` (纯白) | 主体突出 |
| **AI 纯文字 (AI Text)** | `#E3E3E3` (浅灰) | 流畅自然的大白话 |
| **工具成功标志 (SUCCESS)** | `#4CAF50` (亮绿) | 任务推进顺畅，建立信任感 |
| **工具失败标志 (FAILED)** | `#F44336` (明红) | 警示大模型和人类有错发生 |
| **安全阻断 (SECURITY)** | `#FF9800` (明橙) | 白名单拦截，系统安全 |
| **中间思考状态 (THINK)** | `#00BCD4` (青色) | 智能体正在像人类一样“转脑子” |

### 🔄 三、 核心状态机与信号映射 (State & Signal Bridge)

为了完美适配你手写的 `QThread` 多线程架构，智能体和 UI 之间必须通过信号进行如下映射：

```
       [大模型思考中...] ────> 发射 Signal: status_update("🔄 Agent 正在思考...")
             │
             ├──> 分支 A: 发现工具调用 ──> 发射 Signal: status_update("🎯 自动激活工具箱...")
             │                             │
             │                             └──> 多线程并发执行本地 Python / Bash
             │                                   │
             │                                   └──> 成功/失败 ──> 发射 Signal: tool_called(名, 参, 结)
             │
             └──> 分支 B: 给出最终文字 ──> 发射 Signal: ai_message(纯文本答案)

```

#### 💡 视窗事件拦截控制 (Event Filter)

如你之前遇到的 Bug，输入框区域必须严格过滤键盘事件：

* **`Enter` 键：** 触发 `send_message()`，将文本追加到 UI 的 `QTextBrowser` 中，同时唤醒子线程大模型，**并立刻将输入框设为 `setEnabled(False)`（只读状态）**。
* **`Shift + Enter` 键：** 允许用户输入框换行，而不发送消息。
* **智能体工作期间：** 输入框持续锁定，直到内循环触发 `break`（模型吐出最终文本），才重新 `setEnabled(True)` 允许用户继续输入。

### ⌨️ 四、 斜杠增强命令 (Slash Commands)

Claude Code 最好用的特性之一就是快捷命令。在你的 `main-window.py` 发送逻辑中，应该优先拦截以下输入：

* **`/clear`**：一键清空 UI 界面上的聊天记录，并**同步重置**后台的 `messages` 历史记忆（相当于开启全新干净的对话）。
* **`/tools`**：在界面上以漂亮的表格形式，打印出当前系统利用 `inspect` 自动加载上线的 `available_tools` 列表和它们的功能简述。
* **`/status`**：打印当前本地智能体运行的环境上下文（如：当前 `cd` 切换到了哪个项目路径、PaddleX 模型的下载缓存位置、API 的 Key 剩余额度等）。

### 🛡️ 五、 极端情况下的 UI 防御性交互设计

1. **当输出大段代码时：** UI 的渲染组件必须支持 `Markdown` 或 `HTML` 解析（PySide6 的 `QTextBrowser` 原生支持基础 HTML）。大模型吐出的 ````java` 必须被解析包裹在带背景色的矩形框内，防止格式错乱。
2. **当 Bash 工具执行 `Timeout`（超时）时：** 界面上必须以**显眼明橙色**弹出警告，并提供一个 **[强制终止任务]** 的按钮。点击该按钮直接调用 `thread.terminate()` 强行杀死底层子进程，防止本地某些 Gradle 进程死锁导致电脑 CPU 烧满。
3. **工具连续报错触发熔断时：** 界面停止滚动，自动在最下方生成一份 `Agent 任务失败退场报告`，用红色框框起来，明确告诉用户是由于哪几个第三方依赖或配置缺失导致无法继续。
